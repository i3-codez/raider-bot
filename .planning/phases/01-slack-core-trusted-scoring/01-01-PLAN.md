<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Slack -> Bolt HTTP receiver | Untrusted signed requests cross into the runtime here |
| Runtime -> Postgres | Canonical scoring/audit data is persisted here |
| Runtime -> logs | Secrets and database URLs could leak if logging is not redacted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | `src/app/slack.ts` | mitigate | Use Bolt `HTTPReceiver` with `SLACK_SIGNING_SECRET`; do not add custom bypass middleware for Slack routes. |
| T-01-02 | I | `src/config/env.ts` and `src/lib/logger.ts` | mitigate | Validate secrets at startup and redact `DATABASE_URL`, `SLACK_BOT_TOKEN`, and `SLACK_SIGNING_SECRET` from all structured logs. |

</threat_model>
